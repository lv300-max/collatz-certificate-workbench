Capture Signature Theory

Death Coordinate
Every tested seed gets a coordinate:
steps + Ω ratio
Steps = how long it resisted.
Ω ratio = how it resolved.
Seed Classification
Each seed lands in a zone:
FAST DECAY, DECAY, Ω SKIM, CLIMB, STRONG CLIMB, COLLECTOR-CAPTURED, or MONSTER DELAY.
Normalized Delay
steps ÷ log₂(seed) removes size bias. A 10-digit seed with 800 steps ranks higher than a 30-digit seed with 1000 steps. Delay density = steps ÷ digit count.
Peak Timing
Early spike (peak in first third), mid spike, or late spike (peak in last third). Late spike = hardest to capture.
Ω Drift Rate
How fast the even/odd ratio converges toward log₂(3) per 100 steps. Slow drift = longer monster corridor.
Odd/Even Autocorrelation
Are odd steps clustered (long O-runs) or alternating? Max odd run and max even run reveal trajectory texture.
Residue Class of Peak
Which mod-6 and mod-12 class the peak lands in. Tracks whether certain residue classes produce harder peaks.
What we search for
High normalized delay + tight Ω gap + big peak + late spike + long odd-run + slow drift = the monster corridor.

▮ WOBBLE / OCTAVE FRAMEWORK — PUSH HARDER

Collatz is not a pure rise-and-fall story. It is a clean scaling system broken by accumulated +1 wobble: 3on is the octave-copy, b is the wobble ledger, 2c is the braking, B is the wobble cap, and log₂(3) is the speed limit where braking beats kicking.
1 — FALLING ROCKET LAW
Odd step = (3n + 1) / 2k
3n + 1 = kick / pump
/2k = brake / fall
Every odd seed is already inside a falling system.
2 — NO DOUBLE-RISE LAW
Drops can stack: 16 → 8 → 4 → 2 → 1
Raw rises cannot stack twice in a row.
odd × 3 + 1 = even
Every rise immediately pays at least one /2 brake.
3 — PIANO / OCTAVE LANES
n = r + 216q
r = note identity / residue lane
q = octave height
Same residue = same note family at a higher octave.
4 — ONE-LESS DOUBLE LAW
3(2x) + 1 = 2(3x + 1) - 1
double the seed
the 3x part doubles perfectly
but the +1 does not double
Higher octave copy = perfect double minus 1.
5 — CLEAN-COPY VS WOBBLE
Without +1: n → 3n scales cleanly.
With +1: n → 3n + 1 bends the copy.
Tm(n) = (3on + b) / 2c
3on = scale-copy · b = wobble · 2c = brake
6 — OFFSET LEDGER LAW
+1 kicks accumulate into b.
b values: 1, 4, 13, 40, 121
bnext = 3b + 1
The +1s are not random. They are the wobble ledger.
7 — OFFSET CAP LAW
B = ceil(b / (2c - 3o))
b = wobble amount
B = how high wobble can hold the lane up
Above B, if brakes beat kicks, the lane falls.
8 — WOBBLE SPEED LIMIT
log₂(3) ≈ 1.5849625
if c / o > log₂(3), then /2 braking beats ×3 kicking.
1.585 is the wobble speed limit.
9 — BEND-ORBIT RULE
path = bend → brake → bend → brake → peak → descent
Temporary orbit is allowed.
Stable orbit is rejected when:
c/o > 1.585 and n > B
10 — COLLECTOR LANES
40 = collector lane
16 = deeper gravity well
From 1 to 100:
79 hit 40 first · 17 skip to 16 · 4 already below 16: 1,2,4,8
11 — INTEGER AS SYSTEM-STATE
n = value + parity + residue lane + octave
    + offset + brakes + collector route
A number is not lonely. It belongs to a box.
Proof question: do all boxes force capture?
12 — RESIDUE-FIELD MAP
columns = residue lanes
rows = octave height
movement = Collatz rule
+1 = fringe bend · collectors = wells
100 numbers can become 12 columns; all numbers can become 216 lanes.
PRESSURE TEST:
For every residue lane r mod 216, export an exact affine state Tm(n) = (3on + b) / 2c with c/o > log₂(3), gap = 2c - 3o > 0, and cap B = ceil(b/gap).

Then force one of four doors:
1. n > B, so the lane falls below its octave line.
2. the lane hits below-self directly.
3. the lane enters collector 40 or 16.
4. the lane splits into smaller exact lanes until one of the first three doors fires.

Hard target: no lane survives as a permanent bend-orbit once its brake ratio clears 1.585 and its value is above the wobble cap.

▮ SEED COLLAPSE / LANE PRESERVATION FRAMEWORK

Seeds collapse the question. Lanes preserve the every.
Picking one seed collapses the universal question into one route. Once the route enters a collector such as 40 or 16, the test is finished. It cannot wrap back around and prove every number that feeds that collector.
SEED COLLAPSE LAW
A chosen seed collapses the "every number" problem into one trajectory.
Once that trajectory enters a collector, the test ends.
It proves entry, not coverage.
SEED TEST VS LANE TEST
Seed test:
n → path → collector → 1

Lane test:
r mod 2k → exact affine state → descent / collector / merge / split
ONE ROUTE
n = one selected seed
n → T(n) → T2(n) → ... → collector
40 → 20 → 10 → 5 → 16 → 8 → 4 → 2 → 1
Seeds demonstrate.
LANE CERTIFICATE
n = r + 2kq
Tm(n) = (3o n + b) / 2c
gap = 2c - 3o
B = ceil(b / gap)
if gap > 0 and n > B, then Tm(n) < n
Lanes preserve the "every."
PROOF PRESSURE TEST
- Does the lane have exact affine state?
- Does the lane preserve all proof-critical information?
- Does the lane have gap > 0?
- Is B finite and exact?
- Does every member of the lane either fall below itself, hit a collector, merge into a proved lane, or split into proved smaller lanes?
CLAIM BOUNDARY
This framework does not claim proof. It explains why seed testing alone cannot prove Collatz, and why the proof object must be the lane certificate, not the individual seed.
